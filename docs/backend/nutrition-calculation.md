# Расчёт КБЖУ

Документ описывает планируемую логику расчёта пищевой ценности рецептов в проекте **ВкусноТут**.

КБЖУ:

- калории;
- белки;
- жиры;
- углеводы.

## Общая идея

У каждого ингредиента в справочнике хранится пищевая ценность на 100 г.

Пример:

```txt
Творог 2%

на 100 г:
- калории: 80
- белки: 16
- жиры: 2
- углеводы: 3
```

Когда ингредиент добавляется в рецепт, указывается его количество.

Пример:

```txt
Творог 2% — 200 г
```

Система рассчитывает вклад ингредиента в общее КБЖУ рецепта.

## Формула расчёта ингредиента

```txt
Значение ингредиента = значение на 100 г * граммы ингредиента / 100
```

Пример:

```txt
80 ккал * 200 г / 100 = 160 ккал
```

## Расчёт всего блюда

КБЖУ всего блюда считается как сумма КБЖУ всех ингредиентов.

```txt
caloriesTotal = сумма calories всех ингредиентов
proteinsTotal = сумма proteins всех ингредиентов
fatsTotal = сумма fats всех ингредиентов
carbsTotal = сумма carbs всех ингредиентов
```

## Расчёт на 100 г готового блюда

Для расчёта на 100 г нужен вес готового блюда.

Поле в рецепте:

```txt
cookedWeightGrams
```

Формулы:

```txt
caloriesPer100g = caloriesTotal / cookedWeightGrams * 100
proteinsPer100g = proteinsTotal / cookedWeightGrams * 100
fatsPer100g = fatsTotal / cookedWeightGrams * 100
carbsPer100g = carbsTotal / cookedWeightGrams * 100
```

## Почему нужен вес готового блюда

Вес сырых ингредиентов не всегда равен весу готового блюда.

Примеры:

- при запекании часть воды испаряется;
- при варке часть продукта может впитать воду;
- супы содержат много воды, которая влияет на КБЖУ на 100 г, но не добавляет калорий.

Поэтому для корректного расчёта на 100 г нужно хранить вес готового блюда.

## Режимы расчёта

В рецепте планируется поле:

```txt
nutritionCalculationMode
```

Возможные значения:

- `MANUAL`;
- `CALCULATED`.

### MANUAL

КБЖУ вводятся вручную.

Этот режим полезен для рецептов, где пищевая ценность уже известна.

Например:

- рецепт перенесён из книги;
- автор сам посчитал КБЖУ;
- рецепт временно добавлен без полного справочника ингредиентов.

### CALCULATED

КБЖУ рассчитываются автоматически по ингредиентам.

Для этого у каждого ингредиента должны быть:

- КБЖУ на 100 г;
- количество в рецепте;
- вес в граммах.

## Единицы измерения

В рецептах ингредиенты могут указываться не только в граммах.

Примеры:

- яйцо — 1 шт.;
- молоко — 200 мл;
- соль — по вкусу;
- лук — 1 шт.;
- творог — 200 г.

Для расчёта КБЖУ системе всё равно нужны граммы.

Поэтому в `recipe_ingredients` планируется хранить:

```txt
amount
unitId
grams
```

Пользователь видит:

```txt
Яйцо — 1 шт.
```

Система считает:

```txt
1 шт. яйца = 55 г
```

## ingredient_unit_conversions

Для перевода единиц измерения в граммы используется таблица `ingredient_unit_conversions`.

Примеры:

```txt
Яйцо куриное + 1 шт. = 55 г
Лук + 1 шт. = 80 г
Морковь + 1 шт. = 100 г
```

Такой подход позволяет хранить рецепт в удобном виде и при этом считать КБЖУ.

## Особые случаи

### Соль, специи, зелень по вкусу

Для таких ингредиентов можно указывать:

```txt
amount: null
unitId: null
grams: 0
note: по вкусу
```

Или хранить их как обычные ингредиенты без участия в расчёте.

### Вода

Вода влияет на вес блюда, но не добавляет калории.

Для воды можно указывать:

```txt
caloriesPer100g: 0
proteinsPer100g: 0
fatsPer100g: 0
carbsPer100g: 0
```

### Яйцо, белок и желток

Лучше хранить как отдельные ингредиенты:

- Яйцо куриное;
- Яичный белок;
- Яичный желток.

Это нужно потому, что КБЖУ у целого яйца, белка и желтка отличаются.

### Миллилитры

Для жидкостей можно использовать приближённое правило:

```txt
1 мл воды = 1 г
```

Но для других продуктов плотность может отличаться.

Для MVP можно использовать примерные значения.

## Когда пересчитывать КБЖУ

КБЖУ рецепта пересчитывается:

- при создании рецепта;
- при редактировании ингредиентов рецепта;
- при изменении веса готового блюда;
- при переключении режима расчёта на `CALCULATED`.

## Что делать, если изменился ингредиент

Если админ изменил КБЖУ ингредиента, возможны два варианта.

### Простой вариант для MVP

Старые рецепты не пересчитываются автоматически.

Новые рецепты используют новые значения ингредиента.

### Расширенный вариант на будущее

Добавить действие:

```txt
Пересчитать все рецепты с этим ингредиентом
```

Или хранить историю пищевой ценности ингредиентов.

## Почему результат нужно хранить в recipes

Даже если КБЖУ рассчитано автоматически, итоговые значения лучше сохранять в таблице `recipes`.

Плюсы:

- быстрее отдавать список рецептов;
- проще фильтровать;
- проще сортировать;
- не нужно пересчитывать рецепт при каждом запросе;
- изменения ингредиентов не ломают старые рецепты.

## MVP-решение

На первом этапе можно:

- хранить КБЖУ рецепта вручную;
- заложить поля для будущего автоматического расчёта;
- добавить справочник ингредиентов;
- добавить единицы измерения;
- добавить вес ингредиента в граммах.

Автоматический расчёт можно реализовать отдельным этапом.

## Роль единиц измерения в расчёте КБЖУ

Справочник `units` является подготовительным этапом для будущего расчёта КБЖУ рецептов.

На текущем этапе единицы измерения умеют приводиться только к базовой единице своего типа:

```txt
mass   → граммы
volume → миллилитры
count  → штуки
```

Примеры:

```txt
2 кг → 2000 г
1.5 л → 1500 мл
2 ст. л. → 30 мл
3 шт. → 3 шт.
```

Этого недостаточно для полного расчёта КБЖУ, потому что пищевая ценность ингредиентов обычно считается на 100 г.

Перевод в граммы для объёмных и штучных единиц зависит от конкретного ингредиента:

```txt
1 шт. яйца ≠ 1 шт. банана
1 ст. л. масла ≠ 1 ст. л. муки
100 мл воды ≠ 100 мл масла по массе
```

Поэтому для полноценного расчёта позже понадобится отдельная модель связок:

```txt
ingredient_unit_conversions
```

Она будет хранить пересчёты вида:

```txt
ingredientId + unitId → grams
```

Примеры:

```txt
яйцо + шт. → 50 г
мука + ст. л. → 25 г
масло + ст. л. → 14 г
```

## Ингредиенты как основа расчёта КБЖУ

Для будущего расчёта пищевой ценности рецептов добавлен справочник ингредиентов.

Каждый ингредиент хранит КБЖУ на 100 г продукта:

- калории;
- белки;
- жиры;
- углеводы.

На текущем этапе ингредиенты ещё не связаны с рецептами. Они являются базовым справочником, который позже будет использоваться в составе рецепта.

Будущая логика расчёта будет строиться вокруг связки:

```txt
Ingredient → RecipeIngredient → Recipe
```

Пример расчёта для одного ингредиента:

```txt
Куриная грудка: 110 ккал на 100 г
В рецепте используется 200 г
Итог: 110 * 200 / 100 = 220 ккал
```

Для точного пересчёта нестандартных единиц в граммы позже потребуется отдельная таблица конверсий ингредиентов:

```txt
ingredient_unit_conversions
```

Она нужна для случаев, когда вес зависит не только от единицы измерения, но и от конкретного ингредиента:

```txt
1 яйцо
1 банан
1 яблоко
1 ст. л. масла
1 ч. л. соли
```

Примеры будущих конверсий:

```txt
яйцо + шт. → 55 г
банан + шт. → 120 г
оливковое масло + ст. л. → 13.5 г
соль + ч. л. → 5 г
```

## Конверсии ингредиентов в граммы

Для будущего автоматического расчёта КБЖУ добавлена таблица конверсий ингредиентов в граммы:

```txt
ingredient_unit_conversions
```

Она нужна для случаев, когда количество ингредиента в рецепте будет указано не напрямую в граммах.

Например:

- `1 яйцо`;
- `1 банан`;
- `1 яблоко`;
- `1 ст. л. масла`;
- `1 ч. л. масла`.

Универсального значения для единицы `шт.` не существует. Вес зависит от конкретного ингредиента.

Примеры:

```txt
1 яйцо ≈ 55 г
1 банан ≈ 120 г
1 яблоко ≈ 180 г
```

Для объёмных единиц ситуация похожая. В справочнике `units` можно указать, что:

```txt
1 ч. л. = 5 мл
1 ст. л. = 15 мл
```

Но для расчёта КБЖУ нужны граммы. У разных продуктов разная плотность, поэтому для конкретного ингредиента может потребоваться отдельная конверсия:

```txt
1 ст. л. молока ≈ 15 г
1 ст. л. оливкового масла ≈ 13.5 г
```

Будущий расчёт состава рецепта сможет использовать следующую логику:

1. если ингредиент указан в граммах — брать количество напрямую;
2. если ингредиент указан в килограммах — пересчитать через `units`;
3. если ингредиент указан в штуках, ложках или других нестандартных единицах — искать запись в `ingredient_unit_conversions`;
4. после получения граммов рассчитывать КБЖУ на основе данных ингредиента на 100 г.

Этот этап является подготовительным для будущей модели `recipe_ingredients`.

## КБЖУ в базовой модели рецепта

В базовой модели `Recipe` уже добавлены поля КБЖУ:

- КБЖУ на 100 г;
- КБЖУ на весь рецепт;
- вес готового блюда;
- режим расчёта КБЖУ.

На текущем этапе используется режим:

```txt
NutritionCalculationMode.MANUAL
```

Это значит, что значения КБЖУ для seed-рецептов заполняются вручную.

Режим:

```txt
NutritionCalculationMode.CALCULATED
```

заложен на будущее.

Он будет использоваться после добавления состава рецепта:

```txt
recipe_ingredients
```

Будущий автоматический расчёт будет опираться на:

- `ingredients` — КБЖУ ингредиентов на 100 г;
- `units` — единицы измерения;
- `ingredient_unit_conversions` — пересчёт нестандартных единиц в граммы;
- `recipe_ingredients` — состав конкретного рецепта;
- `recipes.cooked_weight_grams` — вес готового блюда.

До появления `recipe_ingredients` расчёт КБЖУ по составу рецепта не выполняется.
