# Миграции и seed-данные

Документ описывает планируемый подход к миграциям и seed-данным в backend-части проекта **ВкусноТут**.

## ORM

Проект использует TypeORM.

## База данных

Проект использует PostgreSQL.

## Основной принцип

Структура базы должна изменяться через миграции.

Настройку `synchronize` нельзя использовать в production.

Планируемый подход:

```txt
synchronize: false
migrations: true
```

## Зачем нужны миграции

Миграции позволяют:

- фиксировать изменения структуры базы;
- повторяемо создавать таблицы;
- применять изменения на разных окружениях;
- откатывать изменения при необходимости;
- видеть историю изменения схемы базы.

## Что должно создаваться миграциями

Через миграции создаются:

- таблицы;
- связи;
- индексы;
- enum-типы;
- ограничения;
- внешние ключи.

## Планируемые первые миграции

### 1. Создание пользователей

Таблица:

```txt
users
```

Основные поля:

- id;
- email;
- passwordHash;
- name;
- role;
- status;
- emailVerifiedAt;
- createdAt;
- updatedAt.

### 2. Создание категорий и тегов

Таблицы:

```txt
categories
tags
recipe_tags
```

### 3. Создание рецептов

Таблицы:

```txt
recipes
recipe_steps
```

### 4. Создание ингредиентов и единиц измерения

Таблицы:

```txt
ingredients
units
ingredient_unit_conversions
recipe_ingredients
```

### 5. Создание избранного

Таблица:

```txt
favorite_recipes
```

## Seed-данные

## Первый супер-админ

В будущем первый супер-админ может создаваться через seed-скрипт.

Пример данных:

```txt
email: admin@example.com
role: SUPER_ADMIN
status: ACTIVE
```

Реальные данные администратора не должны храниться в публичном репозитории.

## На текущем этапе

На текущем этапе миграции и seed-скрипты ещё не настроены.

Этот документ фиксирует планируемый подход.

После подключения TypeORM документ нужно дополнить реальными командами:

- создание миграции;
- запуск миграций;
- откат миграций;
- запуск seed-скриптов.
