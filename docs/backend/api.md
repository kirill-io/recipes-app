# API

Документ будет описывать API backend-части проекта **ВкусноТут**.

На текущем этапе API ещё не реализовано. Ниже зафиксирован планируемый набор эндпоинтов.

## Общие принципы

API должно:

- отдавать только опубликованные рецепты для публичной части;
- не отдавать TypeORM Entity напрямую;
- использовать DTO для входящих данных;
- использовать response DTO или мапперы для исходящих данных;
- поддерживать фильтрацию и пагинацию;
- разделять публичные и защищённые эндпоинты.

## Публичные эндпоинты

### Рецепты

Планируемые эндпоинты:

```txt
GET /recipes
GET /recipes/:slug
```

`GET /recipes` должен поддерживать:

- пагинацию;
- поиск;
- фильтр по категории;
- фильтр по тегам;
- фильтр по КБЖУ в будущем.

`GET /recipes/:slug` должен отдавать:

- основную информацию рецепта;
- категорию;
- теги;
- ингредиенты;
- шаги приготовления;
- КБЖУ;
- изображение.

### Категории

Планируемые эндпоинты:

```txt
GET /categories
GET /categories/:slug
```

### Теги

Планируемые эндпоинты:

```txt
GET /tags
```

## Защищённые эндпоинты

Будут доступны после авторизации.

### Избранное

Планируемые эндпоинты:

```txt
GET /favorites
POST /favorites/:recipeId
DELETE /favorites/:recipeId
```

### Мои рецепты

Планируемые эндпоинты:

```txt
GET /my/recipes
POST /my/recipes
PATCH /my/recipes/:id
POST /my/recipes/:id/submit
```

## Административные эндпоинты

Будут доступны ролям:

- `ADMIN`;
- `SUPER_ADMIN`.

Планируемые разделы:

- управление рецептами;
- управление категориями;
- управление тегами;
- управление ингредиентами;
- управление единицами измерения;
- управление пользователями.

## Модерация

Планируемые эндпоинты:

```txt
GET /admin/recipes/pending
POST /admin/recipes/:id/approve
POST /admin/recipes/:id/reject
```

## На текущем этапе

На текущем этапе файл является заготовкой.

После реализации первых контроллеров нужно будет дополнять документ реальными маршрутами, DTO и примерами ответов.
