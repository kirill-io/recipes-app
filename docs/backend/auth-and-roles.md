# Авторизация и роли

Документ описывает планируемую модель пользователей, авторизации и ролей в проекте **ВкусноТут**.

## Общая идея

Сайт доступен всем пользователям без регистрации.

Гости могут:

- смотреть опубликованные рецепты;
- искать рецепты;
- фильтровать рецепты по категориям и тегам.

Регистрация нужна для личных функций:

- избранные рецепты;
- создание рецептов;
- личный кабинет;
- загрузка своих фотографий к рецептам.

## Роли пользователей

### USER

Обычный зарегистрированный пользователь.

Может:

- добавлять рецепты в избранное;
- удалять рецепты из избранного;
- создавать свои рецепты;
- редактировать свои черновики;
- отправлять рецепты на модерацию;
- видеть статусы своих рецептов.

Не может:

- публиковать рецепты без модерации;
- управлять ингредиентами;
- управлять категориями;
- управлять пользователями.

### TRUSTED_AUTHOR

Доверенный автор.

Может:

- создавать рецепты;
- публиковать свои рецепты без модерации;
- редактировать свои рецепты.

Не может:

- управлять пользователями;
- назначать роли;
- управлять системными настройками.

### MODERATOR

Модератор.

Может:

- смотреть рецепты на модерации;
- одобрять рецепты;
- отклонять рецепты;
- указывать причину отклонения.

Не может:

- назначать роли;
- управлять администраторами;
- менять системные настройки.

### ADMIN

Администратор контента.

Может:

- управлять рецептами;
- управлять категориями;
- управлять тегами;
- управлять ингредиентами;
- публиковать рецепты без модерации;
- модерировать рецепты пользователей.

Не может:

- назначать роль `SUPER_ADMIN`;
- управлять супер-администраторами.

### SUPER_ADMIN

Пользователь с полным доступом.

Может:

- управлять пользователями;
- назначать роли;
- блокировать пользователей;
- управлять рецептами;
- управлять ингредиентами;
- управлять категориями;
- управлять тегами;
- модерировать рецепты;
- выполнять административные действия.

## Статусы пользователей

### ACTIVE

Активный пользователь.

### BLOCKED

Заблокированный пользователь.

Не может:

- входить в аккаунт;
- создавать рецепты;
- добавлять рецепты в избранное.

### DELETED

Пользователь помечен как удалённый.

Физическое удаление пользователя из базы не выполняется, чтобы не ломать связи с рецептами и избранным.

## Способы авторизации

Планируемые способы входа:

- email + password;
- Яндекс;
- ВК;
- Google.

В MVP можно начать с email + password.

На текущем этапе авторизация не является обязательной для первого публичного каталога рецептов, но модель базы проектируется с учётом будущего расширения.

## Подтверждение email

После регистрации пользователь должен получить письмо со ссылкой подтверждения.

Пока email не подтверждён, можно ограничить некоторые действия:

- создание рецептов;
- отправка рецепта на модерацию;
- добавление пользовательских фото.

Избранное можно разрешить сразу или тоже ограничить до подтверждения email.

## OAuth

Для входа через внешние сервисы планируется таблица `user_auth_accounts`.

### Поля

| Поле | Назначение |
|---|---|
| `id` | Уникальный идентификатор |
| `userId` | Пользователь |
| `provider` | Провайдер авторизации |
| `providerUserId` | ID пользователя у провайдера |
| `providerEmail` | Email от провайдера |
| `createdAt` | Дата создания |
| `updatedAt` | Дата обновления |

### Провайдеры

- `YANDEX`
- `VK`
- `GOOGLE`

У пользователя может быть несколько способов входа.

## Модель публикации рецептов

### Обычный пользователь

```txt
DRAFT → PENDING_REVIEW → PUBLISHED
```

Если рецепт отклонён:

```txt
PENDING_REVIEW → REJECTED
```

### Доверенный автор

```txt
DRAFT → PUBLISHED
```

### Админ или супер-админ

Может создавать, редактировать, публиковать и архивировать любые рецепты.

## Первый супер-админ

Первого супер-админа не нужно создавать через публичную регистрацию.

В будущем возможные варианты:

- seed-скрипт;
- ручное создание в базе;
- отдельная команда для создания администратора.

## MVP-решение

На первом этапе можно заложить enum-роли в таблице `users`.

Отдельные таблицы `roles`, `permissions`, `user_roles` пока не нужны.

Если проект сильно вырастет, можно будет перейти к более гибкой системе прав.
