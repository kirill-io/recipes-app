# Категории и теги

Документ описывает подход к категориям и тегам в проекте **ВкусноТут**.

## Основная идея

Категории и теги решают разные задачи.

```txt
Категория = основной раздел сайта
Тег = дополнительный признак рецепта
```

Категория должна отвечать на вопрос:

```txt
Что это за тип блюда?
```

Тег должен отвечать на вопросы:

```txt
Как это готовится?
Чем это удобно?
Для какого случая подходит?
Какая особенность у рецепта?
```

## Почему не стоит копировать структуру книги

В исходных рецептах могут быть авторские разделы:

- минимум усилий;
- блюда в контейнерах;
- блюда в рукаве;
- фастфуд для похудения;
- во время переездов;
- новогодний стол.

Для книги такая структура нормальная, но для сайта удобнее использовать классическую навигацию.

Авторские разделы лучше превратить в теги или подборки.

## Правила категорий

1. У рецепта должна быть одна основная категория.
2. Категории не должны дублировать теги.
3. Категории должны быть понятны пользователю без контекста.
4. Категорий не должно быть слишком много.
5. Категории используются в основной навигации сайта.

## Обновление: реализация категорий

Категории — это основные разделы каталога рецептов.

На текущем этапе категории реализованы как отдельная таблица `categories` и доступны через backend API.

Категории используются для основного разделения рецептов по типу блюда.

Примеры стартовых категорий:

- Завтраки;
- Салаты;
- Супы;
- Основные блюда;
- Десерты;
- Перекусы;
- Напитки.

Категории имеют уникальный `slug`, который в будущем может использоваться в URL страниц каталога.

Пример:

```txt
/categories/breakfasts
/categories/salads
```

Backend API для получения активных категорий:

```txt
GET /{apiPrefix}/categories
```

Если `apiPrefix=api`:

```txt
GET /api/categories
```

Теги будут использоваться позже как дополнительные признаки рецепта: быстро, ПП, без сахара, вегетарианское, острое и другие.

Категория отвечает на вопрос:

```txt
Что это за тип блюда?
```

Тег отвечает на вопрос:

```txt
Какими дополнительными признаками обладает рецепт?
```

## Обновление: стартовый справочник тегов

После категорий добавлен справочник тегов.

Категории и теги выполняют разные задачи.

Категории отвечают на вопрос:

```txt
К какому основному разделу относится рецепт?
```

Примеры категорий:

- завтраки;
- салаты;
- супы;
- основные блюда;
- десерты.

Теги отвечают на дополнительные вопросы:

```txt
Какой это рецепт?
Из чего он?
Как он готовится?
Для какой ситуации подходит?
Есть ли у него диетические особенности?
```

Примеры тегов:

- ПП;
- Быстро;
- Просто;
- Без сахара;
- Низкокалорийное;
- Высокобелковое;
- С курицей;
- С рыбой;
- С творогом;
- Овощное;
- На сковороде;
- В духовке;
- Без выпечки;
- На каждый день;
- Праздничное.

Для MVP принята модель:

- у рецепта будет одна основная категория;
- у рецепта может быть несколько тегов.

Такой подход позволит в будущем строить фильтрацию каталога:

```txt
/api/recipes?category=salads
/api/recipes?tag=bez-sahara
/api/recipes?tag=bystro
```

На текущем этапе связь `Recipe` ↔ `Tag` ещё не создаётся, потому что сущность рецепта ещё не реализована. Связь будет добавлена позже отдельной миграцией при создании рецептов.
