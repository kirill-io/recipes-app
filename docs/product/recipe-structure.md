# Структура рецепта

Документ описывает продуктовую структуру рецепта в проекте **ВкусноТут**.

## Общая идея

Рецепт — главная сущность сайта.

Пользователь должен быстро понять:

- что это за блюдо;
- к какой категории оно относится;
- сколько времени готовить;
- какие ингредиенты нужны;
- как готовить;
- какая пищевая ценность у блюда.

## Основные данные рецепта

У рецепта должны быть:

- название;
- slug для URL;
- краткое описание;
- категория;
- теги;
- изображение;
- время приготовления;
- количество порций;
- сложность;
- ингредиенты;
- шаги приготовления;
- КБЖУ на 100 г;
- КБЖУ на всё блюдо.

## Карточка рецепта

Карточка рецепта в списке должна показывать:

- изображение;
- название;
- категорию;
- основные теги;
- калорийность на 100 г;
- время приготовления;
- признак избранного для авторизованного пользователя.

Пример:

```txt
Паста с тунцом
Основные блюда
89 ккал / 100 г
20 минут
Теги: быстро, много белка
```

## Страница рецепта

Страница отдельного рецепта должна показывать:

- большое изображение;
- название;
- описание;
- категорию;
- теги;
- время приготовления;
- количество порций;
- КБЖУ;
- список ингредиентов;
- шаги приготовления;
- кнопку добавления в избранное.

## Ингредиенты

Ингредиенты должны отображаться в понятном виде:

```txt
Творог 2% — 200 г
Яйцо — 1 шт.
Соль — по вкусу
```

При этом внутри системы ингредиент может иметь дополнительные данные для расчёта КБЖУ:

- справочник ингредиента;
- единица измерения;
- вес в граммах;
- примечание;
- группа ингредиентов.

## Группы ингредиентов

Для некоторых рецептов ингредиенты удобно делить на группы.

Примеры групп:

- Тесто
- Начинка
- Соус
- Заливка
- Глазурь
- Клецки

Пример отображения:

```txt
Тесто:
- Творог 2% — 200 г
- Яйцо — 1 шт.
- Мука — 80 г

Начинка:
- Куриное филе — 300 г
- Лук — 1 шт.
```

## Шаги приготовления

Шаги приготовления должны храниться отдельно.

Пример:

```txt
1. Подготовить ингредиенты.
2. Смешать основу.
3. Выложить в форму.
4. Запекать 30 минут при 180 градусах.
```

Такой подход позволит в будущем сделать:

- режим готовки по шагам;
- чекбоксы выполнения;
- таймеры;
- фото для отдельных шагов.

## Изображения

На первом этапе можно использовать одно поле:

```txt
coverImageUrl
```

Если изображения нет, frontend показывает заглушку.

В будущем можно добавить:

- главное фото рецепта;
- пользовательские фото;
- фото шагов приготовления.

## КБЖУ

Рецепт должен поддерживать два режима КБЖУ:

- ручной ввод;
- автоматический расчёт по ингредиентам.

На первом этапе можно хранить КБЖУ вручную.

В будущем КБЖУ будет рассчитываться по справочнику ингредиентов.

## Рецепты с несколькими вариантами

Некоторые источники могут содержать подборки вида:

- три завтрака;
- три паштета;
- два варианта одного блюда.

Для сайта лучше разделять такие блоки на отдельные рецепты.

Плюсы:

- у каждого рецепта своя карточка;
- у каждого рецепта свои ингредиенты;
- у каждого рецепта своё КБЖУ;
- каждый рецепт можно отдельно добавить в избранное;
- каждый рецепт можно найти через поиск.

Если позже понадобится объединять рецепты, можно добавить подборки.
