# Структура рецепта

Документ описывает продуктовую структуру рецепта в проекте **ВкусноТут**.

## Общая идея

Рецепт — главная сущность сайта.

Пользователь должен быстро понять:

- что это за блюдо;
- к какой категории оно относится;
- сколько времени готовить;
- какие ингредиенты нужны;
- как готовить;
- какая пищевая ценность у блюда.

## Основные данные рецепта

У рецепта должны быть:

- название;
- slug для URL;
- краткое описание;
- категория;
- теги;
- изображение;
- время приготовления;
- количество порций;
- сложность;
- ингредиенты;
- шаги приготовления;
- КБЖУ на 100 г;
- КБЖУ на всё блюдо.

## Карточка рецепта

Карточка рецепта в списке должна показывать:

- изображение;
- название;
- категорию;
- основные теги;
- калорийность на 100 г;
- время приготовления;
- признак избранного для авторизованного пользователя.

Пример:

```txt
Паста с тунцом
Основные блюда
89 ккал / 100 г
20 минут
Теги: быстро, много белка
```

## Страница рецепта

Страница отдельного рецепта должна показывать:

- большое изображение;
- название;
- описание;
- категорию;
- теги;
- время приготовления;
- количество порций;
- КБЖУ;
- список ингредиентов;
- шаги приготовления;
- кнопку добавления в избранное.

## Ингредиенты

Ингредиенты должны отображаться в понятном виде:

```txt
Творог 2% — 200 г
Яйцо — 1 шт.
Соль — по вкусу
```

При этом внутри системы ингредиент может иметь дополнительные данные для расчёта КБЖУ:

- справочник ингредиента;
- единица измерения;
- вес в граммах;
- примечание;
- группа ингредиентов.

## Группы ингредиентов

Для некоторых рецептов ингредиенты удобно делить на группы.

Примеры групп:

- Тесто
- Начинка
- Соус
- Заливка
- Глазурь
- Клецки

Пример отображения:

```txt
Тесто:
- Творог 2% — 200 г
- Яйцо — 1 шт.
- Мука — 80 г

Начинка:
- Куриное филе — 300 г
- Лук — 1 шт.
```

## Шаги приготовления

Шаги приготовления должны храниться отдельно.

Пример:

```txt
1. Подготовить ингредиенты.
2. Смешать основу.
3. Выложить в форму.
4. Запекать 30 минут при 180 градусах.
```

Такой подход позволит в будущем сделать:

- режим готовки по шагам;
- чекбоксы выполнения;
- таймеры;
- фото для отдельных шагов.

## Изображения

На первом этапе можно использовать одно поле:

```txt
coverImageUrl
```

Если изображения нет, frontend показывает заглушку.

В будущем можно добавить:

- главное фото рецепта;
- пользовательские фото;
- фото шагов приготовления.

## КБЖУ

Рецепт должен поддерживать два режима КБЖУ:

- ручной ввод;
- автоматический расчёт по ингредиентам.

На первом этапе можно хранить КБЖУ вручную.

В будущем КБЖУ будет рассчитываться по справочнику ингредиентов.

## Рецепты с несколькими вариантами

Некоторые источники могут содержать подборки вида:

- три завтрака;
- три паштета;
- два варианта одного блюда.

Для сайта лучше разделять такие блоки на отдельные рецепты.

Плюсы:

- у каждого рецепта своя карточка;
- у каждого рецепта свои ингредиенты;
- у каждого рецепта своё КБЖУ;
- каждый рецепт можно отдельно добавить в избранное;
- каждый рецепт можно найти через поиск.

Если позже понадобится объединять рецепты, можно добавить подборки.

## Единицы измерения в структуре рецепта

Для будущего состава рецепта добавлен справочник единиц измерения `units`.

Он нужен для того, чтобы ингредиенты в рецептах можно было указывать в привычном виде:

```txt
200 г творога
1 л молока
2 ст. л. масла
3 шт. яйца
```

На текущем этапе добавлены только сами единицы измерения и их базовая техническая классификация:

- `mass`;
- `volume`;
- `count`.

Справочник `units` пока не решает задачу полного пересчёта ингредиентов в граммы.

Эта задача будет решаться позже через отдельные связи между ингредиентами и единицами измерения.

В будущей модели `recipe_ingredients` единица измерения будет использоваться вместе с количеством ингредиента:

```txt
amount
unitId
ingredientId
grams
displayName
note
groupTitle
```

Поле `grams` в составе рецепта понадобится для расчёта КБЖУ, но его корректное вычисление будет зависеть от справочника ингредиентов и таблицы пересчётов.

## Справочник ингредиентов

В продуктовой модели добавлен справочник ингредиентов.

Ингредиент — это базовый продукт, который в будущем будет использоваться в составе рецепта.

Примеры ингредиентов:

- куриная грудка;
- куриное яйцо;
- творог;
- молоко;
- рис;
- гречка;
- овсяные хлопья;
- картофель;
- банан;
- яблоко;
- помидор;
- огурец;
- морковь;
- оливковое масло.

Для каждого ингредиента хранится пищевая ценность на 100 г:

- калории;
- белки;
- жиры;
- углеводы.

Это позволит позже рассчитывать КБЖУ рецепта автоматически на основании состава, количества ингредиентов и итогового веса блюда.

На текущем этапе ингредиенты ещё не привязаны к рецептам. Связь будет добавляться отдельно через модель состава рецепта.

## Конверсии ингредиентов в граммы

В продуктовую модель добавлены конверсии ингредиентов в граммы.

Они нужны для удобного описания состава рецептов.

Пользователь не всегда будет вводить ингредиенты только в граммах. В рецептах часто используются форматы:

- `1 яйцо`;
- `1 банан`;
- `1 яблоко`;
- `1 ст. л. масла`;
- `1 ч. л. соли`;
- `100 мл молока`.

Для расчёта КБЖУ системе всё равно нужно получить количество ингредиента в граммах. Поэтому для отдельных ингредиентов и единиц измерения хранится правило пересчёта.

Примеры:

```txt
Куриное яйцо + 1 шт. = 55 г
Банан + 1 шт. = 120 г
Оливковое масло + 1 ст. л. = 13.5 г
Оливковое масло + 1 ч. л. = 4.5 г
```

Эта модель не заменяет справочник единиц измерения.

Справочник `units` описывает сами единицы, а `ingredient_unit_conversions` описывает, сколько грамм получается для конкретного ингредиента в конкретной единице.

В будущей модели `recipe_ingredients` это позволит хранить ингредиенты в удобном для пользователя виде, но рассчитывать КБЖУ через граммы.

## Текущая структура рецепта

На текущем этапе рецепт состоит из базовой информации, категории, тегов и шагов приготовления.

### Базовая информация

Рецепт хранит:

- название;
- `slug`;
- краткое описание;
- полное описание;
- изображение;
- время приготовления;
- количество порций;
- сложность;
- статус публикации;
- режим расчёта КБЖУ;
- КБЖУ на 100 г;
- КБЖУ всего;
- вес готового блюда.

### Категория

Каждый рецепт относится к одной основной категории.

Примеры категорий:

- завтраки;
- салаты;
- супы;
- основные блюда;
- десерты;
- перекусы;
- напитки.

Категория используется как основной раздел каталога.

### Теги

Рецепт может иметь несколько тегов.

Теги используются как дополнительные признаки рецепта:

- ПП;
- быстро;
- просто;
- высокобелковое;
- на каждый день;
- с курицей;
- с творогом;
- низкокалорийное;
- овощное.

Связь рецептов с тегами хранится отдельно в `recipe_tags`.

### Шаги приготовления

Шаги приготовления хранятся в отдельной таблице:

```txt
recipe_steps
```

Каждый шаг содержит:

- номер шага;
- описание;
- опциональную ссылку на изображение.

В списке рецептов шаги не отдаются. Они нужны только для детальной страницы рецепта.

### Что ещё не входит в рецепт

На текущем этапе в рецепте ещё нет полноценного состава ингредиентов.

Следующий большой backend-этап:

```txt
recipe_ingredients
```

Он должен связать рецепт с ингредиентами, единицами измерения и граммами, а затем подключить расчёт КБЖУ по составу рецепта.
