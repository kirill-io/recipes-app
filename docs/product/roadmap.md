# План развития проекта

Документ описывает план развития проекта **ВкусноТут**.

## Общий подход

Проект развивается поэтапно.

Основной принцип:

```txt
Сначала рабочий MVP, потом расширение.
```

Архитектура проектируется с учётом будущих возможностей, но реализация выполняется постепенно.

## Этап 1. Базовая настройка проекта

Статус: частично выполнено.

Задачи:

- создать корневой проект;
- создать GitHub-репозиторий;
- создать backend на NestJS;
- настроить запуск backend из корня;
- подключить `.env`;
- настроить базовые инструменты разработки;
- выбрать базу данных;
- выбрать ORM;
- подготовить документацию.

Проделано:

- создан backend на NestJS;
- настроена базовая конфигурация приложения;
- подключён `ConfigModule`;
- подготовлены `.env` / `.env.example`;
- выбрана PostgreSQL;
- выбрана TypeORM;
- подготовлена документация проекта;
- зафиксирован принцип: проектируем шире, реализуем уже.

- конверсии единиц ингредиентов готовы;
- базовая модель рецептов готова;
- теги рецептов готовы;
- шаги рецептов готовы;
- ингредиенты рецептов готовы;
- расчёт КБЖУ готов;
- публичные ручки рецептов работают.

Следующий backend-этап перед переходом к frontend MVP:

- финальная доводка публичного API рецептов;
- фильтрация рецептов по категории;
- фильтрация рецептов по тегу;
- поиск по названию/описанию;
- возможно фильтр по сложности;
- проверка Swagger DTO под frontend;
- финальная проверка seed-данных для 3–5 полноценных рецептов.
