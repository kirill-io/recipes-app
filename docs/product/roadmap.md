# План развития проекта

Документ описывает план развития проекта **ВкусноТут**.

## Общий подход

Проект развивается поэтапно.

Основной принцип:

```txt
Сначала рабочий MVP, потом расширение.
```

Архитектура проектируется с учётом будущих возможностей, но реализация выполняется постепенно.

## Этап 1. Базовая настройка проекта

Статус: частично выполнено.

Задачи:

- создать корневой проект;
- создать GitHub-репозиторий;
- создать backend на NestJS;
- настроить запуск backend из корня;
- подключить `.env`;
- настроить базовые инструменты разработки;
- выбрать базу данных;
- выбрать ORM;
- подготовить документацию.

Проделано:

- создан backend на NestJS;
- настроена базовая конфигурация приложения;
- подключён `ConfigModule`;
- подготовлены `.env` / `.env.example`;
- выбрана PostgreSQL;
- выбрана TypeORM;
- подготовлена документация проекта;
- зафиксирован принцип: проектируем шире, реализуем уже.

## Этап 2. База данных и ORM

Статус: в процессе.

Задачи:

- подключить PostgreSQL;
- подключить TypeORM;
- настроить database config;
- подключить TypeORM в `AppModule`;
- проверить подключение backend к базе данных;
- настроить DataSource;
- настроить миграции;
- описать первые entity;
- создать первую миграцию;
- проверить создание таблиц.

Проделано:

- переустановлен PostgreSQL для чистой локальной разработки;
- выбрана версия PostgreSQL 16;
- создана локальная база данных `recipes_app_dev`;
- pgAdmin 4 оставлен как инструмент просмотра базы;
- зафиксировано, что pgAdmin не нужно запускать при каждой разработке;
- добавлены env-переменные для PostgreSQL;
- создана структура `src/config/database`;
- создан `databaseConfig`;
- создан тип `DatabaseConfig`;
- созданы утилиты чтения env-переменных;
- создан `getTypeOrmConfig`;
- `databaseConfig` добавлен в `configLoaders`;
- настроены публичные экспорты config-разделов;
- принято решение использовать `TypeOrmModule.forRootAsync`;
- принято решение использовать `synchronize: false`.

Ближайшие задачи:

- подключить `TypeOrmModule` в `AppModule`;
- запустить backend и проверить подключение к PostgreSQL;
- исправить возможные ошибки подключения;
- настроить TypeORM DataSource для миграций;
- добавить команды для миграций;
- создать первые entity;
- создать первую миграцию.

Планируемые сущности:

- User;
- Recipe;
- Category;
- Tag;
- Ingredient;
- Unit;
- IngredientUnitConversion;
- RecipeIngredient;
- RecipeStep;
- FavoriteRecipe.

## Что не входит в первый MVP

На первом этапе не реализуются:

- сложная админка;
- OAuth;
- email-подтверждение;
- комментарии;
- рейтинги;
- подписки;
- рекомендации;
- планировщик питания;
- список покупок;
- сложные подборки;
- продвинутая модерация;
- полноценный калькулятор КБЖУ.

Эти возможности заложены в архитектуру, но будут добавляться позже.
