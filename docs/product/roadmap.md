# План развития проекта

Документ описывает план развития проекта **ВкусноТут**.

## Общий подход

Проект развивается поэтапно.

Основной принцип:

```txt
Сначала рабочий MVP, потом расширение.
```

Архитектура проектируется с учётом будущих возможностей, но реализация выполняется постепенно.

## Этап 1. Базовая настройка проекта

Статус: в процессе.

Задачи:

- создать корневой проект;
- создать GitHub-репозиторий;
- создать backend на NestJS;
- настроить запуск backend из корня;
- подключить `.env`;
- настроить базовые инструменты разработки;
- выбрать базу данных;
- выбрать ORM;
- подготовить документацию.

## Этап 2. База данных и ORM

Задачи:

- подключить PostgreSQL;
- подключить TypeORM;
- настроить database module;
- настроить DataSource;
- настроить миграции;
- описать первые entity;
- создать первую миграцию;
- проверить создание таблиц.

Планируемые сущности:

- User;
- Recipe;
- Category;
- Tag;
- Ingredient;
- Unit;
- IngredientUnitConversion;
- RecipeIngredient;
- RecipeStep;
- FavoriteRecipe.

## Этап 3. Seed-данные

Задачи:

- добавить стартовые категории;
- добавить стартовые теги;
- добавить единицы измерения;
- добавить базовые ингредиенты;
- добавить тестовые рецепты.

## Этап 4. Первый API рецептов

Задачи:

- получить список опубликованных рецептов;
- получить рецепт по slug;
- добавить фильтрацию по категории;
- добавить фильтрацию по тегам;
- добавить базовый поиск.

## Этап 5. Frontend

Задачи:

- создать frontend-приложение на Next.js;
- настроить структуру frontend;
- подключить API;
- реализовать страницу списка рецептов;
- реализовать страницу рецепта;
- реализовать фильтры по категориям и тегам.

## Этап 6. Авторизация

Задачи:

- регистрация;
- логин;
- JWT;
- refresh token;
- профиль пользователя;
- подтверждение email.

В будущем:

- вход через Яндекс;
- вход через ВК;
- вход через Google.

## Этап 7. Избранное

Задачи:

- добавить рецепт в избранное;
- удалить рецепт из избранного;
- получить список избранных рецептов пользователя;
- отображать состояние избранного на frontend.

## Этап 8. Пользовательские рецепты

Задачи:

- создание черновика рецепта;
- редактирование своего рецепта;
- добавление ингредиентов;
- добавление шагов приготовления;
- отправка рецепта на модерацию;
- просмотр своих рецептов.

## Этап 9. Модерация

Задачи:

- список рецептов на модерации;
- одобрение рецепта;
- отклонение рецепта;
- причина отклонения;
- статусы рецепта.

## Этап 10. Админка

Задачи:

- управление рецептами;
- управление пользователями;
- управление ролями;
- управление категориями;
- управление тегами;
- управление ингредиентами;
- управление единицами измерения.

## Этап 11. Расчёт КБЖУ

Задачи:

- хранить КБЖУ ингредиентов на 100 г;
- хранить единицы измерения;
- хранить переводы единиц в граммы;
- считать КБЖУ всего блюда;
- считать КБЖУ на 100 г готового блюда;
- поддержать ручной и автоматический режим расчёта.

## Этап 12. Изображения

Задачи:

- загрузка главного изображения рецепта;
- пользовательские фото;
- фото шагов приготовления;
- модерация изображений.

## Что не входит в первый MVP

На первом этапе не реализуются:

- сложная админка;
- OAuth;
- email-подтверждение;
- комментарии;
- рейтинги;
- подписки;
- рекомендации;
- планировщик питания;
- список покупок;
- сложные подборки;
- продвинутая модерация;
- полноценный калькулятор КБЖУ.

Эти возможности заложены в архитектуру, но будут добавляться позже.
