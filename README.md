# ВкусноТут

MVP сайта с рецептами на Next.js, NestJS и PostgreSQL.

## О проекте

**ВкусноТут** — fullstack-приложение для хранения, просмотра и управления рецептами.

Основная идея проекта — сделать удобный каталог рецептов с категориями, тегами, ингредиентами, избранным и расчётом пищевой ценности.

На первом этапе проект разрабатывается как MVP. В дальнейшем планируется расширение функциональности: личный кабинет, пользовательские рецепты, модерация, админка, справочник ингредиентов и автоматический расчёт КБЖУ.

## Установка зависимостей

### Установить зависимости backend

Из корня проекта:

```bash
yarn install:back
```

Команда устанавливает зависимости внутри папки `backend`.

Аналогичный ручной вариант:

```bash
cd backend
yarn install
```

## Запуск проекта

### Запустить backend в режиме разработки

Из корня проекта:

```bash
yarn dev:back
```

Команда запускает NestJS-приложение в режиме разработки.

Аналогичный ручной вариант:

```bash
cd backend
yarn start:dev
```

### Собрать backend

Из корня проекта:

```bash
yarn build:back
```

### Запустить backend после сборки

Из корня проекта:

```bash
yarn start:back
```

## Доступные корневые скрипты

Скрипты находятся в корневом `package.json`.

```json
{
  "install:back": "yarn --cwd backend install",
  "dev:back": "yarn --cwd backend start:dev",
  "start:back": "yarn --cwd backend start",
  "build:back": "yarn --cwd backend build",
  "lint:back": "yarn --cwd backend lint"
}
```

### Описание скриптов

| Скрипт              | Назначение                                   |
| ------------------- | -------------------------------------------- |
| `yarn install:back` | Устанавливает зависимости backend-приложения |
| `yarn dev:back`     | Запускает backend в режиме разработки        |
| `yarn start:back`   | Запускает backend в обычном режиме           |
| `yarn build:back`   | Собирает backend-приложение                  |
| `yarn lint:back`    | Запускает проверку кода backend-приложения   |

## План ближайших этапов

1. Подключить PostgreSQL.

2. Подключить TypeORM.

3. Настроить миграции.

4. Спроектировать и создать базовые сущности:
   - пользователь;
   - рецепт;
   - категория;
   - тег;
   - ингредиент;
   - единица измерения;
   - ингредиент рецепта;
   - шаг приготовления;
   - избранный рецепт.

5. Создать стартовые миграции.

6. Добавить seed-данные:
   - базовые категории;
   - базовые теги;
   - базовые единицы измерения;
   - тестовые ингредиенты.

7. Реализовать первый API для рецептов:
   - получение списка рецептов;
   - получение рецепта по `slug`;
   - фильтрация по категориям и тегам.

8. Создать frontend-приложение на Next.js.

9. Настроить взаимодействие frontend и backend.

10. Реализовать публичную страницу списка рецептов.

11. Реализовать публичную страницу отдельного рецепта.
